PROJECT OVERVIEW
================================================================================
Aromat is a production-ready fragrance encyclopedia built with:
- Framework: Next.js 14 (JavaScript, no TypeScript)
- Styling: Tailwind CSS with Google Fonts
- Database: Local JSON (perfumes.json - 310MB, UTF-8 encoded)
- Design: Monochrome + Gold accent, Serif typography, Luxury aesthetic
- Fonts: EB Garamond (headers) + Inter (body) - Full international support
- International: Full support for Cyrillic, Latin, and extended character sets

STATUS: ✅ PRODUCTION READY - All components functional, dev server running


WHAT WAS BUILT
================================================================================

PAGES (8 files):
1. Homepage (/) - Hero section + Search bar + Random perfume recommender (6 fragrances w/ ISR)
2. Gallery (/gallery) - Responsive 3-column grid with Featured/Top Rated filters
3. Product Page (/perfume/[title]) - Full fragrance details + Scent Pyramid + Reviews
4. Search Results (/search?q=query) - Dynamic results with suggestions
5. Root Component (_app.js) - Global styles wrapper
6. API: Search (/api/search) - Real-time suggestions
7. API: Brands (/api/brands) - Unique brands list
8. API: Fragrances (/api/fragrances) - Featured/top-rated fragrances

COMPONENTS (5 React components):
1. PerfumeCard.js - Fragrance card with image, brand, rating, notes preview (fixed sizing)
2. ScentPyramid.js - Visual pyramid showing Top/Heart/Base notes (cleans text)
3. SearchBar.js - Search input with auto-suggestions dropdown
4. Footer.js - Reusable footer with navigation links
5. cleanText utility - Global text cleaner imported from perfumeData.js

DATA HANDLER (lib/perfumeData.js):
- cleanText(text) - Global text cleaner (removes brackets, quotes, array notation)
- getAllPerfumes(limit) - Fetch fragrances with optional limit
- searchPerfumes(query) - Search by name, brand, or notes (case-insensitive)
- getPerfumeByTitle(title) - Get single fragrance
- getPerfumesByDesigner(designer) - Filter by brand
- getTopRatedPerfumes(limit) - Sort by rating
- getRandomPerfumes(limit) - Random selection for homepage recommender
- getUniqueBrands() - All unique brands
- getUniqueNotes() - All unique fragrance notes
Features: In-memory caching, BOM stripping, UTF-8 support, text normalization

STYLING:
- colors/globals.css - Global CSS + Tailwind directives
- tailwind.config.js - Custom colors (Black #1a1a1a, White, Gold #D4AF37)
- Custom serif fonts for headers (Georgia/Garamond)
- Monochrome palette with single gold accent

DATA STRUCTURE (perfumes.json)
================================================================================

Each fragrance object contains:
{
  "title": "Fragrance Name Brand",
  "designer": "Brand Name",
  "description": "Detailed product description",
  "notes": ["Top Notes", "Heart Notes", "Base Notes", ...],
  "rating": "4.5",
  "reviews": ["Review text 1", "Review text 2", ...],
  "url": "https://source-url.com",
}

File Size: ~310MB
Parsing: Handled by lib/perfumeData.js with caching system
Search: Case-insensitive, matches title, designer, and notes

TROUBLESHOOTING NOTES
================================================================================

Issue: Dev server not starting
Solution: 
  rm -r .next
  npm run dev

Issue: Text showing with brackets and quotes like ['note1', 'note2']
Solution: The cleanText() function automatically handles this globally.
If needed manually, use: cleanText(text) from lib/perfumeData.js

Issue: Cyrillic or special characters not displaying
Solution: Google Fonts import is loaded via @import in styles/globals.css.
Ensure fonts are loading (check Network tab in browser DevTools).
Fonts support: Cyrillic, Cyrillic Extended, Latin, Latin Extended

Issue: Perfume cards have inconsistent heights
Solution: Fixed with flex-grow layout in PerfumeCard.js.
Buttons use mt-auto to stick to bottom. This is permanent fix.

Issue: JSON data shows garbled characters
Solution: File is UTF-8 encoded. BOM stripping happens in getAllPerfumes().
If issues persist, re-encode file to UTF-8:
  $file = "perfumes.json"
  $content = [System.IO.File]::ReadAllText($file, [System.Text.Encoding]::Unicode)
  [System.IO.File]::WriteAllText($file, $content, [System.Text.Encoding]::UTF8)

Issue: Build errors about perfumes.json encoding
Solution: Non-blocking - data still loads correctly. File is UTF-8 and handles BOM strips.

Issue: Large file performance on first load
Solution: First load may be slow due to 310MB JSON. Subsequent loads cached.
Consider database migration for production scale.

Issue: Homepage recommender shows same perfumes each time
Solution: ISR revalidates every 3600 seconds (1 hour). During development,
manual rebuild needed or reset by restarting dev server.
